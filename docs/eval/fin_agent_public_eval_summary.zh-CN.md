# FinSight-Agent 公开评测摘要

本文面向公开读者，概括当前 FinSight-Agent 的质量评测结果、覆盖范围和边界。详细运行记录保留在模型运行记录和工作日志中；本文不堆叠内部运行编号，也不公开云端路径、接口密钥或原始模型响应。

## 评测目标

当前评测不是只判断最终回答是否流畅，而是检查完整研究链路是否可靠：

- 研究负责人是否正确判断问题类型和研究范围。
- 公司范围与关系构建是否知道何时扩展、何时不扩展。
- 证据执行器是否真实调用受控工具，而不是让模型凭记忆回答。
- 覆盖检查是否保留缺口，并在必要时触发二次检索。
- 专家智能体是否只基于限定证据输出结论卡。
- 判断汇总、备忘录写作和校验是否保留证据边界。
- 多轮追问是否能继承状态，同时遵守新的范围约束。
- 后端是否记录耗时、模型调用量、工具轨迹和评测报告。

## 当前评测覆盖

最新完整诊断评测覆盖 20 个真实风格案例：

| 案例类型 | 数量 | 主要考察 |
| --- | ---: | --- |
| 精确查询 | 2 | 单指标查数、数值台账、低成本路径、来源边界 |
| 聚焦回答 | 2 | 单公司基本面、管理层解释、缺口边界 |
| 标准研究备忘录 | 5 | 基本面、市场反应、风险和公司比较 |
| 行业深度研究 | 4 | 产业链、行业背景、关系假设、反证和边界 |
| 多轮追问 | 4 | 上下文继承、范围收缩、旧公司排除、产物复用 |
| 范围决策和缺口升级 | 3 | 何时扩展公司范围、如何保留供应链和非美国来源缺口 |

覆盖行业包括半导体、云计算、银行、能源、消费、医疗、公用事业和人工智能基础设施。

## 当前结果

最新完整诊断评测结果：

| 指标 | 结果 |
| --- | ---: |
| 总通过率 | 20 / 20 |
| 精确查询 | 2 / 2 |
| 聚焦回答 | 2 / 2 |
| 标准备忘录 | 5 / 5 |
| 行业深度 | 4 / 4 |
| 多轮追问 | 4 / 4 |
| 范围决策和缺口升级 | 3 / 3 |
| 平均单案例耗时 | 约 155 秒 |
| 最慢案例耗时 | 约 331 秒 |
| 总模型调用量 | 约 90.6 万 token |

这个结果说明：扩展后的后端诊断链路已经形成可用闭环，能处理查数、聚焦分析、标准备忘录、行业深度、多轮追问、范围决策和缺口升级。

## 评测说明了什么

这轮评测支持以下结论：

- 研究负责人能够根据问题类型决定是否扩展公司范围。
- 公司范围与关系构建能够检查公司清单和关系证据，而不是盲目搜索全库。
- 证据执行器能够接入披露检索、数值台账、市场快照、行业快照、关系图谱和语义召回。
- 专家智能体能够输出有证据边界的结论卡和缺口请求。
- 备忘录写作和最终呈现能够保留“只是假设”“来源缺口”“覆盖边界”等限制。
- 多轮追问能够处理范围收缩，避免旧公司继续污染新问题。
- 后端可以记录任务耗时、工具轨迹、模型调用量和评测结果。

## 评测没有说明什么

这轮评测不应被夸大为生产级服务结果：

- 它是后端诊断门控通过，不是多用户在线服务压测。
- 行业深度和供应链传导问题仍然耗时较高。
- 当前向量检索路径以常驻连接和 CPU 索引为主，尚未成为生产级 GPU 向量索引服务。
- 评测使用已准备的数据产物，公开仓库不包含私有数据、生成索引或模型缓存。
- 系统不会生成个性化投资建议、实时行情或价格目标。
- 部分非美国公司披露、外部供应链确认、实时新闻、电话会和一致预期数据仍未完整纳入主证据。

## 质量门控维度

当前质量体系主要检查以下方面：

| 维度 | 检查内容 |
| --- | --- |
| 问题理解 | 是否识别问题类型、公司范围、研究深度和输出要求 |
| 证据边界 | 是否区分披露、业绩材料、市场、行业和关系来源 |
| 财务分析 | 数值是否来自台账或可追溯证据，期间和单位是否正确 |
| 产业关系 | 是否区分已验证关系、推断关系、假设和来源缺口 |
| 投资判断 | 是否形成有证据支撑的主线，而不是简单证据清单 |
| 风险反证 | 是否保留反证、冲突和关键观察项 |
| 多轮上下文 | 是否遵守新一轮范围，避免旧状态污染 |
| 过程效率 | 是否控制工具调用、模型调用量和二次检索 |
| 权限安全 | 是否阻止智能体越权、私有路径泄露和无来源断言 |
