# Qwen3.6 27B FP8 RTX 5090 Smoke

## Prompt

用户要求在 Qwen3.6-27B-FP8 下载完成后，先做 27B smoke；该步骤必须和新增公司 reviewed-gold case 设计分开，不能把 runtime smoke 当成 benchmark 质量证据。

## Governance Gate

- Hypothesis: Qwen3.6-27B-FP8 可能在 RTX 5090 32GB 上以 FP8 单卡、无 CPU offload 的方式运行，作为后续 27B 对比的候选。
- Decision target: 至少完成一次短 prompt 的 vLLM load + generation，记录加载耗时、生成耗时、显存占用、关键兼容参数和失败条件。
- Ceiling: 只允许判断 deployability，不允许进入 reviewed-gold 或 scored benchmark 质量结论。
- Baseline: 9B 仍是当前 SEC benchmark 主线；历史 Qwen3.5-27B GPTQ 在 4090 上因 CPU offload 过慢，不适合作为主线。
- Efficiency gate: 不使用 CPU offload；先测 4k 单请求，避免直接消耗长上下文。
- Decision label: `diagnostic_only_4k_fp8_smoke`.

## Work Completed

- 拉回本地 JSON/log 证据:
  `reports/quality/qwen36_27b_fp8_5090_vllm_smoke_*.json`
  and

## Results

- Conservative 4k / `gpu_memory_utilization=0.88` failed during vLLM engine init:
  `No available memory for the cache blocks`.
- 4k / `gpu_memory_utilization=0.98` succeeded with no CPU offload.
- Primary successful run:
  `20260520_qwen36_27b_fp8_5090_vllm_smoke_4k_highmem_nothink_template64_r4`
- Load time: 23.2435 seconds.
- Generation time: 2.2086 seconds.
- Output: 29 tokens, 13.1305 tok/s, `finish_reason=stop`.
- Output quality for the smoke prompt was normal after using tokenizer chat-template
  `enable_thinking=False`; plain `/no_think` text did not suppress thinking.
- GPU after load: 31,369 MiB used, 742 MiB free.
- vLLM KV cache: 2.22 GiB available, 20,935 tokens, reported 5.11x maximum concurrency for 4,096-token requests.

## Decision

Qwen3.6-27B-FP8 is deployable for bounded 4k, single-request text generation on the cloud RTX 5090, but only under a high-memory profile (`gpu_memory_utilization=0.98`) with no CPU offload and thinking disabled through the tokenizer template.

It should not replace Qwen9B in the SEC benchmark mainline yet:

- The memory headroom is narrow.
- 8k/32k contexts are not tested.
- Batch throughput is not tested.
- First-request JIT warnings remain.
- New-company cases are still seed-only and lack reviewed context/facts plus exact-value ledger gates.

## Next Step
