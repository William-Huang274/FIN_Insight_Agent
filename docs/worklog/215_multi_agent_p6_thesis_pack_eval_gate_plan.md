From negative ablation `20260601_sector_depth_p4_task_card_smoke_deepseek_v0_5`:

## 3. Root Causes

1. Memo Writer receives a compact judgment inventory, but not a first-class “write this investment memo” object.
2. The verified judgment plan is safe but not optimized for generation. It contains claim cards, outline, thesis plan, constraints, unsupported claims, conflicts, and source notes in a form that still invites copying or repair loops.
3. The verifier requires `memo_thesis_plan` carry-through, which is safe, but it couples a planning object to LLM output shape. Removing it breaks alignment; keeping it can inflate output unless downstream uses a better projection.
4. Eval currently over-trusts final gate pass. It must distinguish:
   - real Memo LLM pass
   - deterministic fallback pass
   - verifier safety pass
   - memo-quality pass
5. Specialist data view improvements increased task understanding, but the final memo path still turns those claims into a short bounded answer rather than a sectioned, decision-useful memo.

## 4. Hypotheses

- H1: A deterministic `memo_thesis_pack` generated by the Aggregator can reduce Memo Writer prompt ambiguity and improve final memo density.
- H2: Memo Writer should consume `memo_thesis_pack` as the primary input and only a minimal claim-ref inventory as audit support.
- H3: The eval gate must fail real-LLM-required cases when Memo Writer falls back, even if the deterministic fallback is safe and verifier passes.
- H4: P6 should optimize “tokens per useful memo section / memo-ready claim”, not just final pass rate.

## 5. Implementation Plan

### P6.1 Eval Gate Fix

- Update `_memo_llm_pass` to inspect summary `llm_routes.memo_writer.route_result.status` before falling back to diagnostics.
- Add a unit test where final state lacks `memo_route_result` but summary says `status=fallback`; expected gate fail.

Gate:

- Unit test fails before fix and passes after fix.

### P6.2 Thesis Pack Contract

- Add `memo_thesis_pack` to Aggregator output with:
  - `core_thesis`
  - `supporting_drivers`
  - `counterarguments`
  - `watch_items`
  - `evidence_strength_map`
  - `source_boundary`
  - `source_claim_refs`
- Carry `memo_thesis_pack` into deterministic memo draft and LLM normalization defaults.
- Keep `memo_thesis_plan` for backward verifier safety during the transition.

Gate:

- Contract tests prove Aggregator emits pack, deterministic memo carries it, and pack refs are subsets of verified ClaimCard refs.

### P6.3 Memo Writer Prompt Projection

- Add `memo_thesis_pack` to compact Memo Writer input.
- Tell Memo Writer to use pack as the primary writing brief and claim cards only as evidence-ref inventory.
- Do not remove `memo_thesis_plan` yet; v0_5 showed this is brittle.

Gate:

- Unit test proves prompt includes `memo_thesis_pack` and keeps `memo_thesis_plan`.
- Real LLM smoke can then compare memo attempts / tokens against v0_4.

### P6.4 Later, Not In This Slice

- Make verifier quality gate require thesis / drivers / counterarguments / watch items for `deep_research`.
- Split `memo-quality pass` from `safety pass` in the real full-chain eval output.
- Add token-efficiency metrics: tokens per memo-ready claim and tokens per useful memo section.

## 6. Current Slice Status

P6.1-P6.3 are implemented and tested.

Decision:

- Keep the P6 thesis-pack projection and renderer/eval gates. It materially improves the final output surface and brings Memo Writer cost back near the P4 baseline while preserving real retrieval and Specialist evidence-quality pass.
- Do not treat this as complete cost optimization. The remaining issue is system-level token cost: v0.3 still reports `high_total_token_cost`, mostly from specialist breadth plus verifier, and Memo Writer still needs one repair after an initial `length` finish.

Next follow-up:

- Tune cost-aware specialist activation and verifier payload separately; do not add more raw evidence to Memo Writer until token-per-useful-claim improves.
